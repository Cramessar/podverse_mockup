Feeds

Retrieve a list of RSS feeds

Returns a paginated list of RSS feeds including each feed’s URL, status, and associated metadata. Supports optional search, limit, and offset parameters for pagination and filtering. Data is sourced from the feed table, joined with feed_flag_status for human-readable status labels.

Authorizations:
Auth0Bearer

integer [ 1 .. 100 ]

Max number of results to return

offset
integer >= 0

Number of results to skip for pagination

Create a new feed

Adds a new RSS feed to the system. This manually inserts a row into the feed table. This will be used only if we want to allow admins to seed new feeds directly. If feed ingestion is handled automatically via external APIs (e.g., Podcast Index), this may be disabled. Required fields include a valid url (must be unique) and a feed_flag_status_id.

Authorizations:
Auth0Bearer
Request Body schema: application/json
required
url
required
string <uri> <= 2083 characters

RSS feed URL

feed_flag_status_id
required
integer
Default: 1

Required status ID from feed_flag_status table (1=active, 2=always-parse, 3=spam, 4=pending-archive, 5=archived, 6=takedown)

parsing_priority
integer [ 0 .. 5 ]
Default: 0

Optional parsing priority (higher gets parsed more often)

container_id
string or null <= 12 characters

Optional ID of the container instance initiating the parse

Retrieve a feed by ID

Fetches complete feed metadata by id.

This includes:

  • url: The RSS feed address
  • feed_flag_status_id: Links to feed_flag_status table (e.g. active, spam)
  • is_parsing: Timestamp set when feed is being parsed
  • parsing_priority: Integer from 0 to 5 used to schedule parsing frequency

Optional joins with channel for UI display, but this route is scoped to the feed table.

Auth0Bearer

required
integer

Update feed status or priority

Updates the feed’s flag status (feed_flag_status_id) or parsing frequency (parsing_priority).

This changes values in the feed table:

  • feed_flag_status_id: e.g. mark as spam, archived, active
  • parsing_priority: integer 0 (lowest) to 5 (highest)

These values are used internally to control parsing behavior and visibility in the system.

Trigger RSS feed re-parse

This sets the is_parsing field on the feed table to the current timestamp, telling the background job to re-fetch and reprocess the feed data.

This is useful if a feed appears stuck or failed to parse previously.

NOTE: Actual parsing happens asynchronously in a worker, not in response to this call.

Authorizations:
Auth0Bearer
path Parameters
feed_id
required
integer

Responses

Local development

http://localhost:8000/admin/feeds/{feed_id}/reparse

Response samples

  • 200
  • 401
  • 404
Content type
application/json
{
  • "message": "Feed reparse has been scheduled."
}

Channels

Retrieve a list of channels

Returns a paginated list of channels with their basic metadata. Supports filtering by text-based search on title, and sorting by id or title. Primary table is channel. Related tables are :

  • medium: used to include the medium type (e.g. podcast, video)
  • feed: used to show the associated RSS feed URL or status (optional).
Authorizations:
Auth0Bearer
query Parameters
limit
integer [ 1 .. 100 ]

Max number of results to return

offset
integer >= 0

Number of results to skip for pagination

search
string

Search term to filter results by name or title

sort_by
string
Enum: "id" "title"

Field to sort results by. Supported fields: id, title.

sort_order
string
Default: "desc"
Enum: "asc" "desc"

Sort order (ascending or descending).

Retrieve a channel by ID

Returns details for a specific channel. Includes associated relationships and metadata such as slug, podcast index ID, feed reference, and medium. Primary table is channel. Related tables are :

  • feed: provides the associated feed URL and parsing status
  • medium: provides the media type of the channel
  • channel_category + category: maps the channel to its categories
  • stats_aggregated_channel: optionally used to show precomputed statistics

Statistics

Retrieve aggregated channel statistics

Returns aggregated stats for channels such as total listens or views. Sorted by highest monthly count (month_current_count DESC). Supports optional filtering by time window (daily, weekly, monthly) and search by channel title or ID. Data is sourced from the channel and stats_aggregated_channel tables.

Items

List all items

Returns a paginated list of items (episodes) with optional filters such as channel ID and publish date range. Also supports search and sorting.

Primary table: item

Related tables:

  • channel: used to filter items by parent channel (channel_id)
  • item_flag_status: used to determine current flag/status of the item
  • stats_aggregated_item: optionally joined to provide aggregated stats like play/download count
Authorizations:
Auth0Bearer
query Parameters
limit
integer [ 1 .. 100 ]

Max number of results to return

offset
integer >= 0

Number of results to skip for pagination

search
string

Search term to filter results by name or title

channel_id
integer

Filter items by their parent channel's ID

status
string
Enum: "active" "pending-archive" "archived" "pending-delete"

Filter items by their flag status (item_flag_status.status)

published_after
string <date-time>

Filter items published after this date

published_before
string <date-time>

Filter items published before this date

sort_by
string
Enum: "pub_date" "title" "id"

Field to sort results by (supports pub_date, title, or id)

sort_order
string
Default: "desc"
Enum: "asc" "desc"

Sort order (ascending or descending)
