Core Admin Tasks:

Log into the system
Search for a podcast
See metadata (podcast, episodes, feed info)
Reparse or flag a feed
View system usage stats
Filter/search podcasts by time, popularity, or name
Export some data (optional)
See logs of what actions were taken

The needed endpoints:

Authenticate and authorize admin users
Expose podcast/feed/episode data
Allow triggering feed actions (reparse, flag)
Return stats (usage, counts, trends)
Return logs of actions
Support search/filter/pagination on large datasets


Auth	-> /api/auth	-> Login and token management
Podcasts ->	/admin/podcasts	 -> Podcasts, searchable and filterable
Feeds	-> /admin/feeds	-> RSS feeds, reparsable, flaggable
Episodes ->	/admin/episodes	-> Episodes under each podcast
Stats ->	/admin/stats	-> Usage metrics and performance data
Logs	-> /admin/logs	-> Audit trail of admin actions

!! Implementation Notes for Auth (for later)
Auth0 handles login UI and token generation
Your backend just needs to verify JWTs on protected routes
Use a decorator like @jwt_required() or middleware
